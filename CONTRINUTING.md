# Guía de Contribución

¡Gracias por considerar contribuir a nuestro proyecto!

## Cómo contribuir

1. Haz fork del repositorio
2. Crea una rama feature (`git checkout -b feature/nueva-funcionalidad`)
3. Haz commit de tus cambios (`git commit -m 'Añade nueva funcionalidad'`)
4. Haz push a la rama (`git push origin feature/nueva-funcionalidad`)
5. Abre un Pull Request

## Convenciones de código

- Sigue el estilo de código existente
- Documenta nuevo código con comentarios claros
- Mantén commits atómicos y mensajes descriptivos

## Reporte de issues

- Usa el template de issues proporcionado
- Describe el problema o sugerencia claramente
- Incluye pasos para reproducir (si aplica)

## Proceso de revisión

- Todos los PRs requieren aprobación de al menos 2 mantenedores
- Los tests deben pasar antes de mergear
- Se pueden solicitar cambios antes de aprobar

## Estructura de ramas

Usamos GitFlow:
- `main`: Versiones estables
- `develop`: Integración continua
- `feature/*`: Nuevas funcionalidades
- `hotfix/*`: Correcciones urgentes
